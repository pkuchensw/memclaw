# Create summary.md - Human-readable report
summary_content = """# Memory Form Selection Research Report

## Executive Summary

This report documents the research findings for optimal memory operation form selection in high-performance computing environments. The analysis covers read, write, copy, and move operations with hardware-specific optimization strategies.

## Methodology

The research analyzed multiple dimensions of memory operations:
- Operation type classification
- Buffer size thresholds
- Memory alignment requirements
- CPU feature detection
- Cache behavior optimization

## Key Findings

### 1. Size-Based Selection Thresholds

| Size Range | Recommended Form | Rationale |
|------------|------------------|-----------|
| 0-64 bytes | Scalar operations | Vector setup overhead exceeds benefit |
| 64-4KB | SSE/AVX vectorized | Optimal vector utilization |
| 4KB-1MB | AVX2/AVX-512 | Maximum throughput |
| >1MB | Streaming/non-temporal | Avoid cache pollution |

### 2. Alignment Considerations

**Critical Alignment Points:**
- 16-byte: Minimum for SSE operations
- 32-byte: Optimal for AVX2
- 64-byte: Cache line boundary, critical for AVX-512
- 4096-byte: Page boundary for huge page utilization

**Strategy:** Use prologue/epilogue scalar code to reach alignment boundaries before vectorized main loop.

### 3. Operation-Specific Recommendations

#### Read Operations
- **Scalar loads**: <64 bytes, random access patterns
- **Vector loads**: 64+ bytes, sequential access
- **Streaming loads**: Large sequential reads with low reuse
- **Prefetch**: Strided access patterns, hide latency

#### Write Operations
- **Standard stores**: General purpose, cache-friendly
- **Non-temporal stores**: >128 bytes, write-combining buffers
- **Streaming stores**: Avoid read-for-ownership overhead

#### Copy Operations
- **Small (<256B)**: Unrolled scalar or REP MOVSB
- **Medium (256B-4KB)**: AVX2 vectorized with aligned loads/stores
- **Large (4KB-1MB)**: AVX-512 with non-temporal stores for destination
- **Huge (>1MB)**: Async DMA or multi-threaded copy

#### Move Operations
- Always check for overlap before selection
- Forward copy: src < dst
- Backward copy: src > dst
- Use memmove abstraction for safety

### 4. Hardware Feature Detection

Dynamic selection based on CPUID:
- SSE4.2: Baseline for vector operations
- AVX2: 256-bit operations, FMA support
- AVX-512: 512-bit operations, mask registers
- ERMS: Enhanced REP MOVSB/STOSB for medium copies
