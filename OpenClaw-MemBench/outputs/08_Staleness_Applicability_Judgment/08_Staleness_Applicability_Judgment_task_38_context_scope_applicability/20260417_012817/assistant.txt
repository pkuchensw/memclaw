### 1. Global Scope ✅ APPLICABLE

### 2. Session Scope ✅ APPLICABLE
- **Staleness Tolerance:** Medium (up to 5s)
- **Rationale:** Session state is isolated to individual user flows. While consistency within a session is important, minor delays (5 seconds) in reflecting cross-session changes are acceptable trade-offs for performance.
- **Recommendation:** Implement session-local caching with 5-second invalidation windows.

### 3. User Scope ✅ APPLICABLE
- **Staleness Tolerance:** Low (up to 1s)

- **Recommendation:** Use write-through caching with 1-second maximum staleness or optimistic UI updates.

### 4. Request Scope ❌ NOT APPLICABLE
- **Staleness Tolerance:** None (0ms)

- **Recommendation:** Staleness constraints should not be applied to request-scoped data.

### 5. Tenant Scope ✅ APPLICABLE
- **Staleness Tolerance:** Medium (up to 10s)
